关于nginx的日志设置

nginx的日志类型包括error.log和access_log
nginx通过log_format来实现这两个log日志的配置，配置语法如下：

我们来查看自己的nginx.conf文件

  • error_log 记录错误了日志的位置和级别(warn)
  • access_log 记录了日志的位置和以main的格式输出
  • log_format main 定义的main打印的格式
    • remote_addr 表示地址
    • remote_user http请求nginx认证的用户名，没有开启认证模块是为空
    • time_local 表示时间
    • request 表示请求行
    • status 发布会的状态
    • body_bytes_sent 返回body的大小
    • http_referer 请求页面的上一级页面
    • http_user_agent 客户端内容
    • http_x_forwarded_for 用户是在哪个IP使用的代理
      我们打开access_log查看记录的内容

我们先来了解一些http协议的知识 http是建立在tcp/ip协议的基础上的，下图是tcp/ip来接的过程，三次握手之后如果支持多次请求就是长连接

|—|:—|
|HTTP协议版本|连接关系|
|HTTP1.0|TCP不能复用|
|HTTP1.1|顺序性TCP复用|
|HTTP2.0|多路复用TCP复用|
现在主流的是http1.1
也就可以得出结论：

  • HTTP请求建立在一次TCP连接基础上
  • 一次TCP请求至少产生一次HTTP请求
